## 一、概述

本文档记录了三种 HTTP 客户端访问 Claude API 时产生的完整 TLS/HTTP 指纹，用于分析 anyrouter 等 WAF 网关的检测维度，并设计对应的模拟方案。

**采集的客户端：**

| 编号 | 客户端 | 说明 |
|------|--------|------|
| A | **httpx 0.28.1** | Anthropic Python SDK 底层 HTTP 库 |
| B | **curl/8.7.1** | 系统 curl（macOS 自带） |
| C | **curl_cffi (Chrome 142)** | 模拟 Chrome 浏览器的 TLS 库 |

## 五、WAF 检测维度分析

### 5.1 WAF 通常检测的维度

| 检测层 | 检测项 | 权重 |
|--------|--------|------|
| **TLS** | JA3/JA4 指纹 | 高 |
| **TLS** | GREASE 支持 | 中 |
| **TLS** | Cipher Suite 顺序和数量 | 中 |
| **TLS** | Extension 列表和顺序 | 中 |
| **TLS** | Supported Groups / Sig Algos | 低 |
| **HTTP/2** | SETTINGS frame 参数 | 高 |
| **HTTP/2** | Akamai H2 Fingerprint | 高 |
| **HTTP/2** | Pseudo Header 顺序 | 中 |
| **HTTP/2** | Priority / Weight | 低 |
| **HTTP** | Header 名称和顺序 | 高 |
| **HTTP** | User-Agent 与 TLS 指纹一致性 | 高 |
| **HTTP** | 特定 Header 存在性 (如 sec-ch-ua) | 中 |
| **行为** | 请求频率和模式 | 高 |
| **行为** | IP 信誉 | 高 |
| **TCP/IP** | TTL / Window Size | 低 |

### 5.2 httpx (原始 SDK) 的 WAF 弱点

1. **ALPN 仅 http/1.1**：不协商 H2，在需要 H2 的网关会失败
2. **无 GREASE**：一眼可识别为非浏览器客户端
3. **User-Agent 暴露**：`python-httpx/0.28.1` 直接标记为 bot
4. **Cipher 特征明显**：Python ssl 模块的默认 cipher 顺序独特
5. **无 ECH/ALPS**：缺少现代浏览器特有扩展
6. **Supported Groups 含 ffdhe**：浏览器通常不包含 ffdhe 系列

### 5.3 curl_cffi Chrome 模式的优势

1. **GREASE 支持**：随机 GREASE 值，符合浏览器行为
2. **ECH 支持**：Chrome 特有的 Encrypted Client Hello
3. **后量子密钥交换**：X25519MLKEM768，最新 Chrome 特征
4. **ALPS 扩展**：Application-Layer Protocol Settings
5. **H2 完整栈**：SETTINGS / WINDOW_UPDATE / Priority 全部匹配 Chrome
6. **JA3/JA4 稳定**：与真实 Chrome 142 一致

## 六、模拟方案

### 6.1 采用方案：curl_cffi Chrome + Anthropic SDK Headers

**TLS 层：** curl_cffi `impersonate="chrome"` → Chrome 142 的完整 TLS 指纹
**HTTP Header 层：** 精确复制 Anthropic Python SDK 0.84.0 的全部 header
**H2 层：** curl_cffi 自动使用 Chrome 的 H2 参数

### 6.2 实现代码

见同目录下 `claude_client.py`

```python
from claude_client import ClaudeClient

with ClaudeClient(
    api_key="sk-ant-xxx",
    base_url="https://your-anyrouter-url",
) as client:
    # 普通请求
    resp = client.messages_create(
        model="claude-sonnet-4-20250514",
        messages=[{"role": "user", "content": "Hello"}],
    )

    # 流式请求
    for event in client.messages_create(
        model="claude-sonnet-4-20250514",
        messages=[{"role": "user", "content": "Hello"}],
        stream=True,
    ):
        print(event)
```
