# AnyRouter OpenCode Bridge 部署文档

## 配置说明

配置文件路径：`~/.config/anyrouter-opencode-bridge/proxy_config.json`

```json
{
    "api_key": "sk-your-anyrouter-api-key",
    "proxy_url": "http://127.0.0.1:7890",
    "use_proxy": false,
    "debug": false,
    "target_base_url": "https://anyrouter.top/v1",
    "host": "127.0.0.1",
    "port": 8765,
    "dashboard_password": "your-dashboard-password",
    "dashboard_secret": ""
}
```

### 关键配置项

| 字段 | 说明 | 注意事项 |
|------|------|----------|
| `api_key` | AnyRouter API 密钥 | 必填，从 AnyRouter 后台获取 |
| `target_base_url` | **上游 API 地址** | **必须是 `https://anyrouter.top/v1`（含 `/v1`）** |
| `use_proxy` | 是否使用 HTTP 代理 | 国内服务器设为 `false`，本地开发设为 `true` |
| `proxy_url` | HTTP 代理地址 | 仅 `use_proxy=true` 时生效 |
| `host` | 监听地址 | 配合 Nginx 反代时用 `127.0.0.1` |
| `port` | 监听端口 | 默认 `8765` |
| `dashboard_password` | Dashboard 登录密码 | 必填，用于 Web 管理界面 |
| `dashboard_secret` | Session 签名密钥 | 留空，首次启动自动生成 |
| `debug` | 调试模式 | 生产环境设为 `false` |

> **`target_base_url` 必须包含 `/v1` 后缀。** 错误配置为 `https://anyrouter.top`（不含 `/v1`）会导致请求打到网站首页而非 API 端点。

## 支持的模型

| 模型 | 类型 |
|------|------|
| `claude-3-5-haiku-20241022` | Haiku |
| `claude-3-5-sonnet-20241022` | Sonnet |
| `claude-3-7-sonnet-20250219` | Sonnet |
| `claude-haiku-4-5-20251001` | Haiku |
| `claude-opus-4-1-20250805` | Opus |
| `claude-opus-4-6` | Opus |
| `claude-sonnet-4-20250514` | Sonnet |

Sonnet/Opus 模型会自动附加 Claude Code 请求格式（`anthropic-beta`、`tools`、`system`、`thinking`）。

## Linux 部署（PM2 方式）

### 前置条件

- Python 3.10+
- PM2（`npm install -g pm2`）
- Nginx（反向代理 + HTTPS）

### 1. 克隆仓库

```bash
cd ~/github
git clone git@zweiteng.tk:server/anyrouter.git
cd anyrouter
```

### 2. 创建虚拟环境

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 3. 配置

```bash
mkdir -p ~/.config/anyrouter-opencode-bridge
cp proxy_config.example.json ~/.config/anyrouter-opencode-bridge/proxy_config.json
```

编辑配置文件，填入 `api_key` 和 `dashboard_password`。

**确保 `target_base_url` 为 `https://anyrouter.top/v1`。**

### 4. 启动（PM2）

```bash
pm2 start .venv/bin/python --name anyrouter -- main.py --host 127.0.0.1 --port 8765
pm2 save
```

### 5. Nginx 反向代理

```nginx
server {
    listen 443 ssl http2;
    server_name anyrouter.your-domain.com;

    ssl_certificate /etc/letsencrypt/live/anyrouter.your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/anyrouter.your-domain.com/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:8765;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # SSE streaming support
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 600s;
        proxy_send_timeout 600s;
    }
}
```

### 6. 验证

```bash
# 健康检查
curl https://anyrouter.your-domain.com/health

# 查看配置（API Key 脱敏）
curl https://anyrouter.your-domain.com/config

# Dashboard
open https://anyrouter.your-domain.com/dashboard
```

## Linux 部署（systemd 方式）

```bash
./linux/install.sh
```

安装脚本会自动创建 venv、安装依赖、部署 systemd 服务。

```bash
# 状态
sudo systemctl status anyrouter-opencode-bridge

# 日志
sudo journalctl -u anyrouter-opencode-bridge -f

# 卸载
./linux/uninstall.sh
```

## macOS 部署

```bash
./macos/install.sh
```

使用 LaunchAgent 管理，登录后自动启动。

## Claude Code 客户端配置

在 Claude Code 或其他客户端中配置：

```
API Base URL: https://anyrouter.your-domain.com/v1
API Key: 任意值（代理会用配置中的 api_key 覆盖）
```

代理会自动：
- 使用 Chrome TLS 指纹（curl_cffi impersonate=chrome）
- 注入 Anthropic Python SDK 0.84.0 请求头
- 为 Sonnet/Opus 模型添加 Claude Code beta 标志和 tools/system
- 过滤请求体，只保留安全字段

## 运维

### 更新代码

```bash
cd ~/github/anyrouter
git pull
pm2 restart anyrouter  # 或 sudo systemctl restart anyrouter-opencode-bridge
```

### 重新加载配置（不重启）

```bash
curl -X POST https://anyrouter.your-domain.com/config/reload
```

### 查看 PM2 日志

```bash
pm2 logs anyrouter --lines 50
```
