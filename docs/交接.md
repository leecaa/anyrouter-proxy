# anyrouter-opencode-bridge 交接说明

## 项目概览
- 本项目是本地代理桥接服务，用于将客户端请求转发到 AnyRouter（`https://anyrouter.top/v1`），并针对 Claude 模型注入工具/系统提示等。
- 运行方式：Python + FastAPI + uvicorn，本地监听 `127.0.0.1:8765`。
- 主要用途：为 OpenCode / CherryStudio 等客户端提供更稳定的 Anthropic 兼容接口访问。

## 目录结构
- `main.py`：核心代理服务与配置逻辑。
- `claude_code_tools.json` / `claude_code_system.json`：Claude Code 工具与系统提示注入模板。
- `proxy_config.example.json`：配置样例。
- `macos/`：macOS LaunchAgent 相关脚本与 plist。
- `requirements.txt`：Python 依赖。

## 配置说明
默认配置文件路径：
- `~/.config/anyrouter-opencode-bridge/proxy_config.json`
- 可通过环境变量 `ANYROUTER_BRIDGE_CONFIG` 覆盖。

关键字段（见 `proxy_config.example.json`）：
- `api_key`：AnyRouter API Key。
- `proxy_url`：上游代理地址（默认 `http://127.0.0.1:2080`）。
- `use_proxy`：是否启用上游代理。
- `debug`：调试日志开关。
- `target_base_url`：上游目标（默认 `https://anyrouter.top/v1`）。

## 服务端点
- 健康检查：`GET /health`
- 读取配置（已脱敏）：`GET /config`
- 热重载配置：`POST /config/reload`
- 代理转发：`/v1/{path}`（支持 `messages` 等 Anthropic 兼容路径）

## macOS 后台服务（LaunchAgent）
安装与运行（用户级，登录后自启）：
- 安装：`./macos/install.sh`
- 状态：`./macos/status.sh`
- 日志：`~/Library/Logs/anyrouter-opencode-bridge.log`
- 卸载：`./macos/uninstall.sh`

脚本说明：
- `macos/install.sh`：创建 venv、安装依赖、写入 LaunchAgent plist 并启动。
- `macos/run.sh`：实际运行入口，设置配置路径并启动 `main.py`。
- `macos/com.anyrouter.opencode.bridge.plist`：LaunchAgent 模板（安装时替换路径）。

## 当前行为要点
- 服务启动时会加载配置与 Claude Code 模板。
- 若 `stdin` 非交互（LaunchAgent 环境），不会触发交互式 setup 向导，避免 `EOFError`。
- `use_proxy=true` 且 `proxy_url` 未运行时，上游连接会失败（常见错误：`All connection attempts failed`）。

## 常见问题与排查
1) 访问 `http://127.0.0.1:8765/health` 失败
- 先确认 LaunchAgent 状态：`./macos/status.sh`
- 看日志：`tail -n 200 ~/Library/Logs/anyrouter-opencode-bridge.log`

2) 返回 `All connection attempts failed`
- 多数是上游代理 `127.0.0.1:2080` 未启动。
- 解决：
  - 启动你的本地上游代理，或
  - 将 `use_proxy` 改为 `false` 并 `POST /config/reload`。

3) 配置修改不生效
- 调用：`curl -sS -X POST http://127.0.0.1:8765/config/reload`

## 建议的验证方式
- `curl -sS http://127.0.0.1:8765/health`
- `curl -sS http://127.0.0.1:8765/v1/messages -H "content-type: application/json" -d '{"model":"claude-3-5-sonnet-latest","max_tokens":64,"messages":[{"role":"user","content":"ping"}]}'`
